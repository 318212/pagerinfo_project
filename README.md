Release notes (v02.180326):
This is Facebook Pager Project allowing user to setup a virtual pager, scraping user's own feed.
By using this app, user do not have to open facebook and still get streamline of information.
The idea and trigger to create this project is inhumane amount of brainrot and shit content on facebook (inhumane also in the context that more than half of the content is either ragebait or engagement farming or God knows what else).
Therefore, creators of this project decided to create scraper that allows to see only relevant, TEXT information. Since, unfortunately in some places in the world, facebook is still used as a communication tool and removing oneself completely from internet can deprive from useful information (rarely, but still).
Pager is supposed to be fully customizable and possible to be tailored to the end user by the end user. (currently it is possible through files and setups but ultimately it should be possible via UI).
In relation to the previous point - depending on the current version of this project it should be actively maintined as facebook loves to change stuff quite often, which can break the scraping (not to mention if something breaks on facebook just because (we all know it's a perfectly reliable company right??))
Process of scraping and therefore using Pager is safeguarded against getting banned buy randomized intervals of scraping as well as attempts to humanize the way it scrapes the posts (it is supposed to look like the user is lazily scrolling their wall).
As far as it technically against Facebook ToS' to use Pager it is in fact not breaking any rules as we scrape our OWN walls as well as get ONLY information that would be available to us anyways.
In short, Facebook Pager can help their users to fully disconnect from all brainrot and losing time on doomscrolling while still having access to potentially important information. We can fully focus on doing something else while our Pager is keeping us updated ONLY on gravely important matters in the text format.<br><br>
